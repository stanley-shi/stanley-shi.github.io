Java Debug optimization:

Sometimes you will face the problem that you need to debug the java application, so you put a lot of debug codes in you code, for example you may use the log4j or something else to print out the debug information.

But one thing you must remember, even if you use the log4j to log your debug information, during the runtime of your code, it still cost some time to run over the debug codes, even if you use log.debug(..) and you set the log level to some higher level like "warn" or something else. For some performance critical application, this maybe a little issue, you have to remove all the debug lines of code, this is definitely the problem.

Today when I ask Jake about this, he told me something:

First you can define a constant such as : public static final boolean debug=true;

then in your code, for the places you want to print the debug information, you can use:

    if(debug){ printdebuginfo(); }

and when deploying the application, just change the definition of the debug to false, then the java compiler will automaticallly remove the printdebuginfo() line of code, and also the if statement will not appear in the binary code. This will definitely save your time and also maintain the performance of your code.

And DO REMEMBER, the constant should be declared as static & final, or this will not work as you expected.

Here is a simple example:

//file name: GC.java

public class GC{

   public static final boolean DEBUG=true;

}

//file name: App.java

public class App{

   public static void main(String[] args){

      if(GC.DEBUG) System.out.println("this is a debug info");

//      doSomething();

   }

}
